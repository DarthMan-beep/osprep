# Категоризирање на датотеки по големина

Напишете **`classify_sizes.sh`** што ги класифицира обичните датотеки во еден директориум според нивната
големина во бајти. (Адаптирано од задача од туторијал на курсот.)

## Барања

1. Скриптата зема **еден аргумент**: директориум.
2. За секоја **обична датотека** директно во тој директориум (игнорирајте ги поддиректориумите),
   испечатете една линија на **stdout**:

   ```
   <filename> <category>
   ```

   каде што `<filename>` е базното име (без патека), а `<category>` се одредува според
   големината на датотеката во бајти:

   | големина (бајти) | категорија |
   |---|---|
   | `0` | `empty` |
   | `1`–`100` | `small` |
   | `101`–`1000` | `medium` |
   | `> 1000` | `large` |

3. Излезните линии мора да бидат **подредени по име на датотека** (растечки).
4. Ако аргументот недостасува или не е директориум, испечатете

   ```
   usage: classify_sizes.sh <dir>
   ```

   на **stderr** и излезете со `1`. Во спротивно излезете со `0`.
5. Мора да помине `shellcheck`.

## Пример

За директориум `data/` што содржи `a` (0 B), `b` (50 B), `c` (500 B), `e` (2000 B):

```console
$ ./classify_sizes.sh data
a empty
b small
c medium
e large
```

> **Совети:** итерирајте со `for f in "$dir"/*`; прескокнувајте не-датотеки со
> `[ -f "$f" ] || continue`; земете големина со `stat -c%s "$f"`; земете го базното име со
> `basename "$f"`. Спроведете го излезот од јамката низ `sort` за да се подреди по име.
