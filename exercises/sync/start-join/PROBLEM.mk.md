# Стартувај нишка и чекај ја

Претходната задача го напиша `run()` на нишка. Сега ја правиш другата страна: ја
**стартуваш** нишката и **чекаш** да заврши.

Дадена е готова `Worker` (прочитај ја во *Дадени датотеки* подолу). Нејзиниот `run()` спие
кратко, па ја пресметува сумата `1..100` во `result`. Спиењето е важно: ако го прочиташ
`result` прерано, нишката сè уште не завршила и ќе видиш `0`.

## Задача

Дополни ја **`Launcher.computeWithThread()`** така што:

1. ја **стартува** работничката нишка, потоа
2. **чека** да заврши, потоа
3. го враќа `w.result`.

```java
public class Launcher {
    public static long computeWithThread() throws InterruptedException {
        Worker w = new Worker();
        // стартувај ја, чекај ја, па врати w.result
        return w.result;
    }
}
```

Оценувачот ја повикува `computeWithThread()` и очекува **5050**.

> **Совети:** `w.start()` го лансира `run()` на нова нишка и се враќа *веднаш* — **не** чека.
> За да чекаш додека нишката заврши, повикај `w.join()`. Значи редоследот е: `start()`,
> потоа `join()`, па прочитај `result`. Замисли да го тргнеш `join()` — ќе вратиш `0`
> бидејќи работникот сè уште спие. (Никогаш не повикувај `w.run()` директно; тоа би го
> извршило на *тековната* нишка, не на нова.)
