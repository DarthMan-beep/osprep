# Трансфер на пари без deadlock

`Bank` чува неколку сметки. `transfer(from, to, amount)` мора да ги заклучи **двете** сметки
за да не се изгубат пари поради race. Почетниот код ги заклучува **по редослед на
аргументите** — прво `from`, потоа `to`:

```java
public void transfer(int from, int to, int amount) {
    synchronized (lock[from]) {
        synchronized (lock[to]) {
            balance[from] -= amount;
            balance[to]   += amount;
        }
    }
}
```

Ова е безбедно за нишки, но **запира (deadlock)**: ако една нишка прави `transfer(0, 1, …)`
додека друга прави `transfer(1, 0, …)`, првата го зема `lock[0]` и чека `lock[1]`, додека
втората го зема `lock[1]` и чека `lock[0]` — ниедна не може да продолжи (кружно чекање).

## Твоја задача

Направи `transfer` да биде **и** безбеден за нишки **и** без deadlock.

## Барања

- Конкурентни трансфери во спротивни насоки мора да **завршат** (без deadlock).
- **Вкупната** сума пари низ сите сметки се зачувува (двете салда се ажурираат атомично).

> **Совет — доследен редослед на заклучување го крши циклусот:** секогаш заклучувај ги
> двете брави во ист глобален редослед без оглед на насоката, на пр. по индекс на сметка:
> `int first = Math.min(from, to), second = Math.max(from, to);` па заклучи `first` потоа
> `second`. (Една глобална брава околу целиот трансфер исто така работи, само со помалку
> конкурентност.)
