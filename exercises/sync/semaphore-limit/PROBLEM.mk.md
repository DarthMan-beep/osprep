# Ограничи конкурентност со семафор

Понекогаш не сакаш *меѓусебно исклучување* (еден по еден) — сакаш да дозволиш **до N**
нишки во секцијата истовремено (на пр. само N конекции до база, N паралелни преземања).
За тоа служи **броячкиот семафор**.

`LimitedResource.access(work)` ја извршува дадената `work`, но почетниот код ја извршува
**без ограничување** — сите нишки влегуваат одеднаш.

## Твоја задача

Направи `access` да дозволува **најмногу `LIMIT` нишки** да извршуваат `work.run()`
истовремено:

```java
public class LimitedResource {
    public static final int LIMIT = 3;

    public void access(Runnable work) throws InterruptedException {
        work.run();   // сè уште нема ограничување — поправи го ова
    }
}
```

## Барања

- Никогаш повеќе од `LIMIT` нишки внатре во `work.run()` истовремено.
- Секој повикувач на крај влегува (без deadlock, без гладување).

> **Совет:** создади еден споделен `java.util.concurrent.Semaphore` со `LIMIT` дозволи.
> `acquire()` пред извршување на работата и `release()` потоа — стави го `release()` во
> `finally` за да не се изгуби дозвола дури и ако работата фрли исклучок:
> `sem.acquire(); try { work.run(); } finally { sem.release(); }`.
