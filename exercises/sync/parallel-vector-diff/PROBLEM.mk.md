# Паралелна сума на разлики

Дадени се два цели низи `a` и `b` со иста должина. Пресметај:

```
result = Σ |a[i] - b[i]|
```

Работата се дели на **`threads`** работни нишки — секоја обработува дел од низите.
Почетниот код ги собира парцијалните суми од сите нишки во едно **споделено** поле
`total`, **без синхронизација**, па нишките се тркаат на `total += …` и дел од
ажурирањата се губат — резултатот излегува премал.

## Твоја задача

Имплементирај `VectorDiff.sumOfAbsDiff(int[] a, int[] b, int threads)` така што ја враќа
**точната** сума користејќи `threads` нишки — без изгубени ажурирања, за која било
големина на низа и број на нишки.

```java
public class VectorDiff {
    public static long sumOfAbsDiff(int[] a, int[] b, int threads)
            throws InterruptedException {
        // подели ги низите на `threads` делови, собери го секој дел во нишка,
        // па комбинирај ги парцијалните суми во вкупната.
    }
}
```

## Барања

- Ја враќа точната сума, доследно низ повеќекратни извршувања.
- Работи кога должината на низата **не се дели** со `threads`, и кога `threads == 1`.

> **Совет (идиоматската поправка):** дај ѝ на секоја нишка **сопствен** акумулатор (на пр.
> слот во `long[] partial` индексиран по бројот на нишката, или thread-local променлива).
> Секоја нишка пишува само во својот слот — па нема споделено пишување за тркање — а по
> `join()` главната нишка ги собира парцијалите. (Заклучување на споделена `total` при
> секое собирање исто така работи, но е многу побавно.)
