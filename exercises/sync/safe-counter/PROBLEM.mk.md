# Безбеден бројач за нишки

Класата **`Counter`** чува цел број и нуди `increment()` и `get()`. Почетната верзија **не
е безбедна за нишки**: кога повеќе нишки повикуваат `increment()` истовремено, некои
зголемувања се губат (*race condition*), бидејќи `value++` всушност се три чекори — читај,
додај еден, запиши назад — кои можат да се испреплетат.

## Твоја задача

Направи ја `Counter` **безбедна за нишки** така што ако **K нишки секоја повика
`increment()` M пати**, конечната вредност да биде **точно K × M**, секој пат.

```java
public class Counter {
    private int value = 0;

    public void increment() {
        value++;          // не е безбедно: читај-измени-запиши може да се испреплете
    }

    public int get() {
        return value;
    }
}
```

## Барања

- По конкурентните зголемувања, `get()` мора да ја врати точната сума (без изгубени
  ажурирања), доследно низ повеќекратни извршувања.
- Не смее да настане deadlock или закочување.

> **Совет:** наједноставната поправка е критичната секција да стане меѓусебно исклучива со
> клучниот збор `synchronized` (на методите, или `synchronized(this){ … }` блок). И
> `increment()` и `get()` треба да користат иста брава. (Алатки од `java.util.concurrent`
> како `ReentrantLock` или `AtomicInteger` исто така би работеле.)
