# Производител–потрошувач ограничен бафер

Имплементирај **безбеден за нишки ограничен бафер** — срцето на проблемот
производител–потрошувач. Повеќе производители повикуваат `put(item)`, повеќе потрошувачи
`take()`, делејќи бафер со фиксен капацитет.

Почетниот код е расипан: нема заклучување ниту чекање, па губи елементи кога е полн, враќа
ѓубре кога е празен и се корумпира при конкурентност.

## Твоја задача

Направи ја `BoundedBuffer` точна:

```java
public class BoundedBuffer {
    public BoundedBuffer(int capacity) { ... }
    public void put(int item) throws InterruptedException { ... }
    public int take() throws InterruptedException { ... }
}
```

## Барања

- **`put` блокира** додека баферот е **полн**; **`take` блокира** додека е **празен**.
- Безбеден за нишки: секој ставен елемент се зема **точно еднаш** (без губење, без
  дупликати, без корупција).
- Без deadlock.

> **Совет — мониторски шаблон:** направи ги `put` и `take` `synchronized` на истиот објект.
> Чувај го условот во **`while`** циклус (не `if` — нишка може да се разбуди лажно):
> `while (full) wait();` во `put`, `while (empty) wait();` во `take`. По промена на баферот,
> повикај `notifyAll()` за да ја разбудиш другата страна. (`Queue<Integer>` како
> `LinkedList` е практичен за складирање.)
