# RUN при градење, CMD при стартување

Оваа задача е за двете инструкции што најчесто се мешаат:

- **`RUN`** извршува команда **додека сликата се гради** — резултатот се вградува во сликата
  како слој.
- **`CMD`** ја поставува командата што се извршува **кога контейнерот ќе стартува** од
  сликата.

## Задача

Напиши **`Dockerfile`** што:

1. **При градење**, создава датотека `/status.txt` што го содржи зборот `READY` — со
   **`RUN`**.
2. **Кога контейнерот се извршува**, ја печати таа датотека на екран — со **`CMD`**.

Извршувањето на сликата мора да испечати:

```
READY
```

Која било мала Linux база работи (на пр. `debian:12-slim`).

> **Совети:** за да ја создадеш датотеката при градење, изврши shell команда што запишува
> текст во датотека (`echo ... > /status.txt`). За да испечатиш датотека кога контейнерот
> стартува, командата е `cat`. Значи: една `RUN` линија што ја прави датотеката и една
> `CMD` линија што ја `cat`-ира.
