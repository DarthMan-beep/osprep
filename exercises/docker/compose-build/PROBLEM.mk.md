# Compose: гради од Dockerfile

Во претходната compose задача услугата користеше готова слика (`image: nginx:alpine`).
Овој пат, Compose треба **сам да ја изгради сликата** од `Dockerfile`.

Build контекстот (прикажан во *Дадени датотеки* подолу) содржи работен **`Dockerfile`** и
**`page.html`**. Dockerfile-от стартува мал веб сервер на порта **8000** во контейнерот.

## Задача

Напиши **`compose.yaml`** со една услуга (било кое име) што:

- **гради** од `Dockerfile` во тековниот директориум (наместо да користи `image:`), и
- ја објавува портата **8000** од контейнерот на **порта 8087 на домаќинот**.

Оценувачот извршува `docker compose up --build` и проверува дека
`http://localhost:8087/page.html` содржи `built by compose`.

> **Совет:** за да градиш наместо да повлекуваш, замени го `image:` со **`build:`**.
> Наједноставната форма е `build: .` — `.` значи „користи го Dockerfile во овој директориум
> како build контекст.“ Сè друго (`ports:`) е исто како претходно.
