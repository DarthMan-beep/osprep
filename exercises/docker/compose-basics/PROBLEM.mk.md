# Compose: само слика и порта

Docker **Compose** опишува еден или повеќе контейнери во една датотека, `compose.yaml`,
за да можеш да стартуваш сè со една команда (`docker compose up`). Да почнеме со
апсолутниот минимум: **една услуга, готова слика и објавена порта.**

## Задача

Напиши **`compose.yaml`** со една услуга со име **`web`** што:

- ја извршува сликата **`nginx:alpine`**, и
- ја објавува портата **80** од контейнерот на **порта 8086 на домаќинот**.

Без build, без volumes, без environment — само тие две поставки. Оценувачот извршува
`docker compose up` и проверува дека `http://localhost:8086/` ја враќа nginx welcome
страницата.

## Формата на compose датотека

```yaml
services:          # најгорно ниво: мапа од имиња на услуги
  <name>:          # една услуга
    image: ...     # која слика да се извршува
    ports:
      - "HOST:CONTAINER"
```

> **Совет:** `ports` е **листа** (забележи го `-`), и секој запис е цитиран
> `"host:container"` стринг — тука, домаќин `8086` мапиран на контейнер `80`. Ова е
> compose еквивалент на `docker run -p 8086:80 nginx:alpine`.
