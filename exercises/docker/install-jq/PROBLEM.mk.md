# Алатката што недостасува

Build контекстот има две датотеки:

- **`data.json`** — мала JSON датотека што опишува производ.
- **`show.sh`** — shell скрипта што ја чита `data.json` и го печати името на производот. Тоа
  го прави со **`jq`**, командно-линиски JSON процесор.

Напиши **`Dockerfile`** така што извршувањето на сликата печати точно:

```
Fridge
```

## Замката

`show.sh` работи добро на машина каде `jq` е инсталиран — но slim базната слика **не носи
`jq`**. Ако само ги копираш датотеките и ја извршиш скриптата, ќе видиш нешто како
`jq: not found`. Твојот Dockerfile мора да обезбеди алатката да е достапна внатре во сликата
пред да се изврши скриптата.

> Базираните на Debian/Ubuntu слики инсталираат пакети со нивниот менаџер на пакети;
> релевантниот тука се вика `jq`. (`show.sh` ги очекува датотеките на `/app`.)
