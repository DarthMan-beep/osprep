# Сервирај датотека преку HTTP

Контејнеризирај едноставен сервис за хостирање датотеки (како catalog host од вежбата).
Build контекстот вклучува **`products.txt`**:

```
Fridge;$600;5
TV;$1500;6
Microwave;$200;10
```

Напиши **`Dockerfile`** што сервира датотеки од **`/srv/catalog`** користејќи го вградениот
HTTP сервер во Python, слушајќи на **порта 9000** во контејнерот. Серверот се стартува со:

```
python3 -m http.server 9000 --directory /srv/catalog
```

Кога оценувачот ќе ја стартува твојата слика (објавувајќи ја портата), ова барање мора да
успее:

```
GET /products.txt  ->  HTTP 200, телото содржи "Fridge"
```

## Барања

1. `FROM` Python базна слика.
2. `COPY` `products.txt` во `/srv/catalog`.
3. `EXPOSE 9000` (ја документира портата).
4. `CMD` го стартува HTTP серверот прикажан погоре.

> **Совет:** `CMD` како JSON низа, по еден токен на елемент:
> `CMD ["python3", "-m", "http.server", "9000", "--directory", "/srv/catalog"]`.
> Не ја објавуваш портата во Dockerfile — `EXPOSE` само ја документира; оценувачот ја
> мапира со `-p` кога го стартува контејнерот.
