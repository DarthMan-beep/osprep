# Bash: први чекори

Никогаш не си пишувал скрипта? Почни тука. До крајот на оваа страница ќе имаш напишано и
извршено вистинска Bash скрипта. Трае околу 10 минути.

## Што *е* Bash скрипта?

Bash скриптата е едноставно **обична текстуална датотека што ги содржи истите команди што
би ги напишал во терминалот**. Кога ќе ја извршиш датотеката, школката ја чита **од врвот
кон дното** и ја извршува секоја линија, една по една. Тоа е сè — нема компилирање, нема
магија.

Значи, ако можеш да напишеш `echo hello` во терминал, веќе можеш да напишеш скрипта.

## Твојата прва скрипта

Создади датотека со име `greet.sh` со овие две линии:

```bash
#!/bin/bash
echo "Hello!"
```

- **Линија 1** е *shebang*. Му кажува на системот „користи ја програмата `bash` за да ја
  извршиш оваа датотека.“ Секогаш нека биде првата линија.
- **`echo`** печати текст на екранот.

Сега изврши ја. Два чекора — направи ја извршлива еднаш, па изврши ја:

```console
$ chmod +x greet.sh     # „оваа датотека смее да се извршува“
$ ./greet.sh            # изврши ја (./ значи „датотеката токму тука“)
Hello!
```

🎉 Тоа е работна скрипта.

## Коментари

Сè по `#` е белешка за луѓе и Bash го игнорира:

```bash
#!/bin/bash
# Оваа скрипта поздравува. (оваа линија не прави ништо при извршување)
echo "Hello!"
```

## Променливи — чување вредност

Променливата е именувано кутиче што чува вредност:

```bash
#!/bin/bash
name="Ana"
echo "Hello, $name!"     # печати: Hello, Ana!
```

Три правила што ги збунуваат сите на почеток:

1. **Без празни места околу `=`** при доделување. `name="Ana"` работи; `name = "Ana"` **не**.
2. За да ја *искористиш* вредноста, стави **`$`** пред неа: `$name`.
3. **Завиткај ја во двојни наводници**: `"$name"`. (Ова спречува изненадувања кога
   вредноста има празни места — подоцна ќе видиш зошто.)

## Користење влез — аргументи

Кога извршуваш скрипта, можеш да ѝ предадеш зборови. Првиот е `$1`, вториот `$2`, итн.:

```bash
#!/bin/bash
echo "Hello, $1!"
```

```console
$ ./greet.sh World
Hello, World!
$ ./greet.sh Ana
Hello, Ana!
```

Значи `$1` е „што и да напишал корисникот како прв збор.“

## Носење одлука — `if`

Скриптите стануваат корисни кога можат да избираат. Тука проверуваме дали корисникот
навистина ни дал име:

```bash
#!/bin/bash
if [ -z "$1" ]; then
  echo "Please give me a name."
else
  echo "Hello, $1!"
fi
```

Прочитано на глас: *„**ако** `$1` е празно, испечати потсетник, **инаку** поздрави го.“*

- `[ -z "$1" ]` е **тест**: `-z` значи „дали е ова празно?“.
- Празните места во `[ ... ]` се **задолжителни** — `[` всушност е команда, и ѝ требаат
  аргументите одделени со празни места.
- `if ... then ... else ... fi` — забележи дека завршува со `fi` (`if` наназад).

```console
$ ./greet.sh
Please give me a name.
$ ./greet.sh Ana
Hello, Ana!
```

## Сега знаеш доволно за да почнеш

Веќе можеш: да напишеш датотека, да ја извршиш, да користиш променливи, да читаш влез и да
гранеш со `if`. Тоа ги покрива првите неколку задачи.

**Следно:**
- Пробај ја задачата **Скрипта за поздравување** — користи токму овие идеи.
- Кога ќе го сакаш целиот алат (циклуси, `case`, низи, заменување команди, сите тест
  оператори, наводници во длабочина), прочитај го водичот **Bash скриптирање**.

> Една навика вредна за усвојување веднаш: секогаш завиткувај ги променливите во наводници
> (`"$1"`, `"$name"`) и ставај празни места во `[ ]`. Тие две навики ги спречуваат
> најчестите почетнички грешки — а и линтерот на оценувачот (`shellcheck`) ќе те потсетува
> за нив.
