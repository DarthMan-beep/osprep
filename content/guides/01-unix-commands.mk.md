# Unix команди

## Школката (shell): што всушност е таа

Школката (`bash`, `sh`, `zsh`, …) е **интерпретер на команди**. Нејзината срцевина е
бесконечна јамка вообичаено наречена **REPL** (read–eval–print loop, читај–процени–испечати):

1. Испечати промпт (`$` за обичен корисник, `#` за root).
2. **Прочитај** една линија од влезот.
3. **Парсирај** ја: подели ја на зборови, прошири променливи, прошири џокери (wildcards), постави
   пренасочувања и цевки.
4. **Процени**: пронајди ја програмата именувана со првиот збор и `fork()`+`exec()`
   изврши ја (или изврши *вградена* команда како `cd` директно во школката), чекај таа да
   заврши.
5. Испечати го резултатот / следниот промпт и повтори ја јамката.

Општа форма на команда:

```
command_name -options arguments
```

- **Првиот збор** е она што се извршува. Школката го бара или како вградена команда,
  или како датотека пронајдена со пребарување на директориумите наведени во `PATH`
  променливата на околината (провери со `echo $PATH`).
- **Опциите** (наречени и flags, switches) го менуваат однесувањето. Тие се
  **чувствителни на големина на букви** и конвенционално почнуваат со `-`. Опциите од една буква можат да бидат
  **комбинирани**: `ls -a -l` е идентично со `ls -al` или `ls -la`. Долгите опции
  користат `--`, на пример `ls --all`.
- **Аргументите** се работите врз кои дејствува командата (обично имиња на датотеки).

### Како да добиете помош

- `man command` — целосниот прирачник. Внатре во man страница нотацијата `[ ]` означува
  **опционален** дел, а `...` означува нешто што може да се **повторува**. Притисни `q` за излез,
  `/word` за пребарување.
- `command --help` — брз преглед на употребата за повеќето GNU алатки.
- `type command` — ти кажува дали нешто е вградена команда, алијас или датотека
  на дискот (и каде).

> **Ментален модел што треба да го носите низ сè подолу:** речиси секоја алатка чита
> поток од бајти од **стандардниот влез** и запишува поток од бајти во
> **стандардниот излез**. Работата на школката е да *ги поврзе тие потоци заедно*
> (датотеки, цевки, терминали). Откако ова ќе го усвоите, пренасочувањето и цевките
> престануваат да бидат магија.

## Датотеки и директориуми

Unix има едно **вкоренето дрво** што почнува од `/`. Патеката е **апсолутна** ако
почнува со `/` (се разрешува од коренот) и **релативна** во спротивно (се разрешува
од вашиот тековен директориум). Две специјални имиња постојат во секој директориум: `.`
(овој директориум) и `..` (родителот).

| Команда | За што служи | Клучни опции |
|---|---|---|
| `pwd` | печати го работниот директориум (каде се наоѓаш) | `-P` разреши симболички врски |
| `cd DIR` | смени директориум | `cd ..` родител · `cd ~` / `cd` домашен · `cd -` претходен директориум |
| `ls` | излистај содржина на директориум | `-a` вкл. скриени · `-l` долг · `-h` човечки читливи големини · `-t` по време · `-R` рекурзивно |
| `cp SRC DST` | копирај датотека | `-r` рекурзивно · `-p` зачувај метаподатоци · `-i` прашај пред препишување · `-s` направи симболичка врска |
| `mv SRC DST` | премести / преименувај | `-i` прашај · `-n` никогаш не препишувај |
| `rm NAME` | отстрани (избриши) | `-r` рекурзивно · `-f` присилно (без прашање, игнорирај непостоечки) |
| `touch FILE` | создај празна датотека / ажурирај временски печат | `-c` не создавај ако недостасува |
| `cat FILE` | печати или спои датотеки | `-n` нумерирај линии · `-b` нумерирај непразни · `-s` собери низи од празни линии |
| `wc FILE` | брои | `-l` линии · `-w` зборови · `-c` бајти · `-m` знаци |
| `cut` | извлечи колони | `-c LIST` знаци · `-f LIST -d C` полиња по разделувач |
| `find PATH` | прошетај низ дрвото на директориуми, тестирај го секој запис | `-name` · `-type` · `-mtime` · `-size` · `-exec` |

### Како функционираат

**`pwd`/`cd`** — школката чува „тековен работен директориум“ по процес.
`cd` е *вградена* команда (мора да биде — таа мора да ја менува сопствената состојба на школката; а
посебна програма би можела да го смени само сопствениот директориум и потоа да заврши). `cd -`
скока назад во претходниот директориум.

**`ls -l`** — долгиот формат е оној што мора да можете да го прочитате на прв поглед:

```console
$ ls -l
-rw-r--r--  1 ana  staff   1280 Jun 18 09:14 report.txt
drwxr-xr-x  2 ana  staff   4096 Jun 17 22:01 src
```

Поле по поле: `-`/`d` е **типот** (`-` обична датотека, `d` директориум,
`l` симболичка врска), потоа **9 бита за дозволи** (види Дозволи подолу), **бројот на врски**,
**сопственик**, **група**, **големина во бајти**, **време на измена**, **име**.

**`cp -r` наспроти датотеки** — без `-r`, `cp` одбива да копира директориум. `cp -p`
ги задржува оригиналните временски печати и дозволи; обично `cp` ги ресетира на „сега“
и ја применува вашата umask.

**`rm` нема враќање назад.** `rm -rf dir` тивко брише цело подстебло. Класичната
катастрофа е неочекуван празен простор: `rm -rf / home/me/tmp` (забележи го празниот простор по
`/`) се обидува да го избрише целиот датотечен систем. Секогаш двојно проверувај ги џокерите (подолу)
пред да ги комбинираш со `rm -f`.

**`cat`** буквално кон**кате**нира (спојува): `cat a b c` ги испраќа a, потоа b, потоа c во
stdout. Со една датотека едноставно ја испишува; тоа е честата секојдневна употреба.

**`wc`** — решен пример:

```console
$ wc poem.txt
   3   12   67 poem.txt
   │    │    └─ bytes
   │    └────── words
   └─────────── lines
$ wc -l poem.txt
3 poem.txt
```

**`cut`** — извлекување колони. Два режима: по позиција на знак (`-c`) или по
разграничено поле (`-f` со `-d`).

```console
$ echo 'ana:1001:/home/ana' | cut -d: -f1,3
ana:/home/ana
$ echo 'abcdef' | cut -c2-4
bcd
```

Стапица: `cut -f` третира **еден единствен знак** како разделувач и **не**
ги собира повторените разделувачи. Ако колоните се одделени со *низи* од празни простори, `cut`
ќе се однесува погрешно — користи `awk` наместо тоа, кој по дифолт се дели на низи од празни простори.

### `find` — прошетувачот низ дрвото

`find` почнува од `PATH` и **рекурзивно го посетува секој запис**, применувајќи синџир
од **тестови**; записите што ги поминуваат сите тестови активираат **акција** (дифолт акцијата е
`-print`). Тестовите се поврзани со логичко И (AND) од лево кон десно.

| Тест / акција | Значење |
|---|---|
| `-name '*.c'` | името одговара на џокер (стави во наводници за да *школката* не го прошири прво) |
| `-iname` | име нечувствително на големина на букви |
| `-type f` / `-type d` | обична датотека / директориум |
| `-mtime -5` | изменето пред помалку од 5 дена (`+5` = повеќе од, `5` = точно) |
| `-size +2000c` | поголемо од 2000 бајти (`k`, `M` исто така работат; гол број = блокови од 512 бајти) |
| `-exec CMD {} \;` | изврши CMD врз секое погодување; `{}` е името на датотеката, `\;` го завршува |
| `-exec CMD {} +` | изврши CMD еднаш со многу имиња на датотеки групирани заедно (побрзо) |

```console
$ find / -name '*alpha*' -print           # every file with "alpha" in its name
$ find . -type f -mtime -1                 # files changed in the last 24h
$ find . -name core -exec rm -f {} \;      # find core dumps and delete each
$ find . -name '*.log' -size +1M           # log files bigger than 1 MB
```

Замка: во `-name '*.c'` наводниците се суштински. Без наводници, **школката** би го
проширила `*.c` против *тековниот* директориум пред `find` воопшто да се изврши, што би го расипало
рекурзивното пребарување.

## Дозволи — `chmod`

Секоја датотека има **сопственик** (корисник) и **група**, и сет од 9 бита за дозволи
поделен во три класи — **u**ser (сопственик), **g**roup (група), **o**ther (други) — секоја со
три бита: **r** (read, читање), **w** (write, запишување), **x** (execute, извршување). `ls -l` ги прикажува како
`rwxr-x---`.

### Што значи rwx зависи од тоа дали е датотека или директориум

| Бит | Кај **датотека** | Кај **директориум** |
|---|---|---|
| `r` | читај ја содржината | **излистај** ги имињата внатре (`ls`) |
| `w` | измени ја содржината | создај/избриши/преименувај записи внатре |
| `x` | изврши ја како програма | **влези / премини** низ него (`cd`, користи патека низ него) |

Клучни последици:

- `x` без `r` кај директориум: можеш да направиш `cd` во него и да пристапиш до датотека *ако
  веќе го знаеш нејзиното име*, но не можеш да направиш `ls`.
- `r` без `x` кај директориум: `ls` може да ги прикаже имињата, но всушност не можеш да стигнеш до
  датотеките (добиваш „Permission denied“ при пристап).
- За да избришеш датотека ти треба `w` кај нејзиниот **директориум**, а не кај самата датотека —
  бидејќи бришењето ја менува листата на записи на директориумот.

### Два начини за поставување дозволи

**Симболички** — `chmod [who][op][perms]`. `who` ∈ `u g o a` (a = сите),
`op` ∈ `+ - =` (додај / отстрани / постави точно), `perms` ∈ `r w x`.

```console
$ chmod u+x script.sh      # give the owner execute
$ chmod go-w shared.txt    # remove write from group and others
$ chmod a=r notes.txt      # everyone: read only, nothing else
```

**Октални** — секоја класа е една октална цифра, формирана од нејзините три бита прочитани како
бинарно `rwx`:

| rwx | бинарно | октално |
|---|---|---|
| `rwx` | 111 | 7 |
| `rw-` | 110 | 6 |
| `r-x` | 101 | 5 |
| `r--` | 100 | 4 |
| `-wx` | 011 | 3 |
| `--x` | 001 | 1 |
| `---` | 000 | 0 |

Значи три цифри = сопственик, група, други:

```console
$ chmod 640 f      # -rw-r-----  owner rw, group r, others nothing
$ chmod 755 f      # -rwxr-xr-x  owner all, others read+enter/execute
$ chmod 600 f      # -rw-------  private file
```

Зошто ова функционира: `6` = `4+2` = `r + w`; `5` = `4+1` = `r + x`; `7` = `4+2+1` =
сè. Едноставно ги собираш вредностите на местата (r=4, w=2, x=1).

> Совет: `755` е нормалниот режим за директориуми и програми (секој може да ги користи,
> само сопственикот може да ги менува); `644` е нормален за обични датотеки.

## Процеси

**Процес** е програма во извршување со уникатен **PID**. Школката го стартува секоја
надворешна команда како процес-дете и нормално **чека** на неа (преден план).

```console
$ command &        # run in the BACKGROUND: shell prints [job] PID and returns at once
$ ps               # processes in this terminal
$ ps -ef           # EVERY process, full listing (UID, PID, PPID, start, command)
$ ps -ef | grep nginx
$ jobs             # background/stopped jobs started by THIS shell
$ fg %1            # bring job 1 back to the foreground
$ kill PID         # ask a process to terminate (sends SIGTERM = signal 15)
$ kill -9 PID      # force kill (SIGKILL = 9, cannot be caught/ignored)
$ pidof -s bash    # PID(s) of a program by name; -s = single PID
```

### Како функционираат позадинските задачи

`&` ѝ кажува на школката **да не чека**. Процесот продолжува да се извршува и веднаш го добиваш
промптот назад. `jobs` ги излистува оние стартувани од тековната школка со броеви на
задачи (`%1`, `%2`); `fg`/`bg` ги преместуваат меѓу преден план и позадина, а
`Ctrl-Z` ја суспендира тековната задача на предниот план.

### Сигнали и `kill`

`kill` не мора нужно „да убие“ — таа **испраќа сигнал**. По дифолт е `SIGTERM`
(15), учтиво „те молам исклучи се“ што програмата може да го обработи (на пр. да ги испразни датотеките).
`kill -9` испраќа `SIGKILL`, што кернелот веднаш го спроведува и програмата
не може да го фати — користи го само кога `SIGTERM` се игнорира.

### Виртуелниот датотечен систем `/proc`

`/proc` не е вистински диск; кернелот **ги синтетизира** овие датотеки при читање, изложувајќи
живата состојба на кернел/процес како текст. За процес со PID *N*:

| Патека | Содржина |
|---|---|
| `/proc/<pid>/status` | состојба читлива за луѓе, меморија (VmRSS, VmSize), UID, нишки |
| `/proc/<pid>/statm` | големини на меморија **во страници** — помножи со големината на страница |
| `/proc/<pid>/maps` | секој мапиран меморски регион (код, heap, stack, библиотеки) |
| `/proc/<pid>/cmdline` | точната командна линија (одделена со NUL) |

```console
$ cat /proc/self/status | head -3
$ getconf PAGESIZE          # usually 4096 bytes → multiply statm fields by this
$ pmap -x <PID>             # formatted memory map with sizes per region
```

Замка: броевите во `statm` се во **страници**, а не во бајти. Вредност `1000` со
страница од 4096 бајти значи ~4 MB. Секогаш множи со `getconf PAGESIZE`.

## Пренасочување и цевки

Секој процес почнува со три отворени потоци идентификувани со **дескриптори на датотеки**:

| Поток | FD | Дифолт |
|---|---|---|
| стандарден влез (**stdin**) | 0 | тастатурата |
| стандарден излез (**stdout**) | 1 | терминалот |
| стандардна грешка (**stderr**) | 2 | терминалот |

Пренасочувањето ѝ кажува на школката **повторно да поврзе дескриптор на датотека** *пред*
командата да се изврши. Самата команда не знае ниту се грижи — таа само го чита FD 0 и
запишува во FD 1/2.

| Оператор | Значење |
|---|---|
| `> file` | stdout → датотека (**создај/препиши**) |
| `>> file` | stdout → датотека (**додај на крај**) |
| `< file` | stdin ← датотека |
| `2> file` | stderr → датотека |
| `2>&1` | stderr → каде и да оди stdout во моментот |
| `&> file` | и stdout и stderr → датотека (bash) |
| `\| cmd2` | stdout од левиот → stdin на десниот (**цевка**) |
| `tee file` | копирај го stdin во *датотека* **и** проследи го понатаму до stdout |

```console
$ ls -l > list.txt                 # overwrite list.txt with the listing
$ echo "line" >> log.txt           # append one line
$ sort -rn < nums.txt > sorted.txt # read from one file, write to another
$ make 2> errors.txt               # keep only the error messages
$ cmd > out.txt 2>&1               # both streams into out.txt (order matters!)
$ cmd > /dev/null 2>&1             # discard everything
```

### Како всушност функционира цевката

`A | B` прави школката да создаде **бафер за цевка во кернелот**, потоа да ги изврши **A и B во
исто време**: stdout (FD 1) на A е поврзан на крајот за запишување, stdin (FD 0) на B
на крајот за читање. Бајтите што A ги произведува течат директно во B без воопшто да го
допрат дискот. Ако B е бавен, A се блокира кога баферот ќе се наполни (back-pressure, противпритисок); кога
A завршува и го затвора својот крај, B гледа крај-на-влез.

```console
$ ls -l | grep '^d' | wc -l        # count subdirectories: list → keep dir lines → count
$ ls | tee files.txt | wc -l       # save the list AND count it in one pass
```

Замки:

- `cmd > out.txt 2>&1` работи, но `cmd 2>&1 > out.txt` **не** ги пренасочува
  грешките во датотеката — пренасочувањата се применуваат **од лево кон десно**, така што во моментот
  на `2>&1`, stdout сè уште беше терминалот.
- `>` ја кратко (truncates) датотеката во моментот кога командата стартува. `sort f > f` ќе
  **ја испразни f пред sort да ја прочита** — никогаш не пренасочувај во датотека што истовремено ја
  читаш.
- Само **stdout** тече низ цевката; **stderr** сè уште оди до терминалот
  освен ако не додадеш `2>&1`.

## Globbing (џокери за имиња на датотеки)

Globbing-от го прави **школката**, *пред* командата да се изврши. Школката го проширува
шаблонот во листата на **постоечки имиња на датотеки што одговараат** и ги проследува тие како
посебни аргументи. Командата никогаш не го гледа `*` — таа само добива листа од
имиња. (Спореди со regex-ите на `grep`, кои самиот `grep` ги интерпретира врз текст.)

| Шаблон | Одговара на |
|---|---|
| `?` | точно еден знак |
| `*` | било која низа од знаци (вклучувајќи и ниту еден) |
| `[abc]` | еден од наведените знаци |
| `[2-4]` | еден знак во опсегот |
| `[^0-9]` / `[!0-9]` | еден знак што **не** е во сетот |
| `{a,b}` | проширување со загради: произведува `a` и `b` (не е џокер — работи дури и ако не постои датотека) |

```console
$ echo *.txt                 # show what the glob expands to before running anything
$ rm *[^0-9]                 # remove files whose name does NOT end in a digit
$ mv ab* new/                # move everything starting with "ab"
$ cp file{1,2,3}.c backup/   # expands to file1.c file2.c file3.c
```

Замки:

- Ако џокерот не одговара на **ништо**, bash (по дифолт) го проследува шаблонот
  **буквално**. `ls *.xyz` без совпаѓања извршува `ls *.xyz` и дава грешка „No such
  file or directory“ — не извршува обично `ls`.
- `*` **не** одговара на водечка точка. Скриените датотеки (`.bashrc`) се прескокнуваат
  освен ако точката не ја напишеш експлицитно (`.[!.]*`) или поставиш `dotglob`.
- Проширувањето со загради `{a,b}` се случува дури и без датотеки што одговараат и **не** е
  џокер; тоа е чисто генерирање текст.

## Филтрирање: grep · sed · awk

Овие три го читаат влезот **линија по линија** и одлучуваат, по линија, што да прават. Тие се
‘рбетот на цевководите (pipelines). Регуларните изрази подолу се **Basic Regular
Expressions (BRE, основни регуларни изрази)** освен ако не е наведено поинаку.

### grep — пронајди линии што одговараат

**За што служи:** ги печати линиите од својот влез што одговараат на regex (пребарување).

**Како функционира:** го чита влезот по една линија; ако линијата содржи совпаѓање
за шаблонот, ја печати **целата линија** (по дифолт). Таа *не* менува
ништо.

Основна regex синтакса: `^` почеток на линија · `$` крај на линија · `.` било кој еден знак ·
`*` нула-или-повеќе од претходниот · `[ ]` класа на знаци · `\` избегни (escape) специјален
знак.

| Опција | Ефект |
|---|---|
| `-v` | инвертирај: печати ги линиите што **не одговараат** |
| `-i` | нечувствително на големина на букви |
| `-c` | печати само го **бројот** на линии што одговараат |
| `-n` | стави го бројот на линија пред секое погодување |
| `-l` | печати само ги **имињата** на датотеките што содржат совпаѓање |
| `-r` | рекурзирај во директориуми |
| `-E` | проширен regex (`+`, `?`, `|`, `()` без обратни коси црти) |

```console
$ grep '^From ' mailbox          # lines that START with "From "
$ grep -v '^#' config            # everything EXCEPT comment lines
$ grep -i unix notes.txt         # match Unix, UNIX, unix, …
$ grep -rn TODO src/             # every TODO under src/, with file:line
$ grep -E '^[0-9]{3}-[0-9]{4}$' phones   # lines that are exactly NNN-NNNN
```

Решен пример:

```console
$ cat fruit.txt
apple
banana
Apricot
$ grep -i '^a' fruit.txt
apple
Apricot
```

### sed — стрим уредувач (stream editor)

**За што служи:** трансформира текст во лет — најчесто **барај-и-замени**.

**Како функционира:** ја чита секоја линија во „pattern space“ (простор за шаблон), ги применува твоите команди
за уредување и (по дифолт) го печати резултатот. По дифолт **не** ја менува
датотеката; излезот оди во stdout. Главниот работник е командата за замена
`s/old/new/flags`.

| Команда | Значење |
|---|---|
| `s/re/repl/` | замени го **првото** совпаѓање на секоја линија |
| `s/re/repl/g` | замени ги **сите** совпаѓања на линијата |
| `s/re/repl/gi` | сите совпаѓања, нечувствително на големина на букви |
| `/re/d` | избриши ги линиите што одговараат на re |
| `/re/p` | печати ги линиите што одговараат (користи со `-n`) |
| `-n` | потисни го автоматското печатење (така само `p` дава излез) |
| `-i` | уреди ја датотеката **на место** (опасно; направи резервна копија) |

```console
$ sed 's/Aple/Apple/g' f.txt     # fix every "Aple" → "Apple"
$ sed '/^#/d' config             # drop all comment lines
$ sed -n '/error/p' log          # behave like grep: print only matches
$ sed -n '2,5p' file             # print only lines 2 through 5
```

Решен пример:

```console
$ echo 'color colour color' | sed 's/colou\?r/COLOR/g'
COLOR COLOR COLOR        # (using -E or \? in BRE; matches both spellings)
```

Замка: `s///` без `g` го менува само **првото** појавување по линија. И
`sed -i` ја препишува датотеката без враќање назад — тестирај без `-i` прво.

### awk — процесор на полиња/записи

**За што служи:** обработка ориентирана на колони и мали извештаи — кога размислуваш
во термини „поле 1, поле 3, сума на поле 2“.

**Како функционира:** awk го чита влезот по еден **запис** (линија) и го дели
на **полиња**. Програмата е листа од `pattern { action }` правила: за секоја линија,
секое правило чиј `pattern` е точен ја извршува својата `action`. Изостави го шаблонот за да се изврши на
секоја линија; изостави ја акцијата и дифолтот е „печати ја линијата“.

| Вградено | Значење |
|---|---|
| `$0` | целата линија |
| `$1`, `$2`, … | прво, второ, … поле |
| `NF` | број на полиња на оваа линија |
| `NR` | број на тековната линија (запис) |
| `-F C` | постави го разделувачот на полиња на C (дифолт: низи од празни простори) |
| `BEGIN { }` | се извршува еднаш **пред** било каков влез |
| `END { }` | се извршува еднаш **по** сиот влез |

```console
$ awk '{ print $1, $3 }' data            # print columns 1 and 3
$ awk -F: '{ print $1 }' /etc/passwd     # usernames (passwd is colon-separated)
$ awk '$3 > 75 { print $0 }' grades      # whole line where field 3 > 75
$ awk 'END { print NR }' file            # line count (like wc -l)
$ awk 'BEGIN{c=0} /^ *$/{c++} END{print c}' f   # count blank lines
$ awk '{ s += $2 } END { print s }' nums         # sum of the 2nd column
```

Решен пример:

```console
$ cat scores.txt
ana 90
bob 60
cyd 78
$ awk '$2 >= 75 { print $1 }' scores.txt
ana
cyd
```

Забелешка: за разлика од `cut -f`, дифолт поделбата на awk собира **низи** од празни простори, така што
правилно ги обработува колоните дополнети со повеќе празни простори.

### sort / uniq / tr

**`sort`** — подредува линии. По дифолт лексикографски (така `10` доаѓа пред `9`).

| Опција | Ефект |
|---|---|
| `-n` | нумеричко подредување (`9` пред `10`) |
| `-r` | обратно |
| `-k N` | подреди по поле N |
| `-u` | отфрли ги дупликат линиите по подредувањето |

**`uniq`** — собира само **соседни** дупликат линии. Ова е класичната
стапица: таа **не** наоѓа дупликати што не се еден до друг, па речиси секогаш
ги цевководиш `sort | uniq`. `-c` става пред секоја линија нејзиниот број; `-d`
ги прикажува само дуплираните линии.

**`tr`** — преведи или избриши **знаци** (чита само од stdin, без аргумент за име на датотека).
`tr 'a-z' 'A-Z'` префрла во големи букви; `-d` брише сет; `-s` собира низи од
знак во еден.

```console
$ sort -n nums.txt                    # 2, 9, 10  (numeric)
$ sort names.txt | uniq -c            # frequency of each name
$ echo 'Hello World' | tr 'a-z' 'A-Z' # HELLO WORLD
$ echo 'a    b' | tr -s ' '           # collapse spaces → "a b"
$ cat file | tr -d '\r'               # strip Windows carriage returns
```

Решен пример — најчести зборови по фреквенција:

```console
$ cat words.txt
red
blue
red
red
blue
$ sort words.txt | uniq -c | sort -rn
      3 red
      2 blue
```

> **Вежбање:** Unix вежбите ти дозволуваат да ги градиш овие цевководи и да го провериш
> точниот излез. Обиди се да го предвидиш излезот на секоја команда *пред* да ја извршиш — тоа е
> она што го бара испитот од тебе.
